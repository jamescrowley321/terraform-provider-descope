# descope_password_settings (Resource)

Manages password authentication settings for a Descope project. This is a singleton resource — every project has exactly one password policy. Creating this resource configures the settings; destroying it removes the resource from Terraform state but does not delete the settings from the project.

## Example Usage

```hcl
resource "descope_password_settings" "example" {
  enabled          = true
  min_length       = 12
  lowercase        = true
  uppercase        = true
  number           = true
  non_alphanumeric = true
  expiration       = true
  expiration_weeks = 26
  lock             = true
  lock_attempts    = 5
}
```

## Schema

### Optional

- `enabled` (Boolean) Whether password authentication is enabled. Defaults to `true`.
- `expiration` (Boolean) Whether passwords expire after a set number of weeks. Defaults to `false`.
- `expiration_weeks` (Number) Number of weeks before a password expires. Valid range: 1–999. Defaults to `1`.
- `lock` (Boolean) Whether to lock accounts after a number of failed login attempts. Defaults to `false`.
- `lock_attempts` (Number) Number of failed login attempts before locking the account. Valid range: 2–10. Defaults to `2`.
- `lowercase` (Boolean) Whether passwords must contain a lowercase letter. Defaults to `false`.
- `min_length` (Number) Minimum password length. Valid range: 4–64. Defaults to `8`.
- `non_alphanumeric` (Boolean) Whether passwords must contain a non-alphanumeric character. Defaults to `false`.
- `number` (Boolean) Whether passwords must contain a number. Defaults to `false`.
- `reuse` (Boolean) Whether to prevent password reuse. Defaults to `false`.
- `reuse_amount` (Number) Number of previous passwords that cannot be reused. Valid range: 1–50. Defaults to `1`.
- `uppercase` (Boolean) Whether passwords must contain an uppercase letter. Defaults to `false`.

### Read-Only

- `id` (String) The ID of this resource.
